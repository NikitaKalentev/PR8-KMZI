# Электронная подпись по ГОСТ Р 34.10-2012 (Python)
Данный проект реализует российский стандарт электронной цифровой подписи ГОСТ Р 34.10-2012 на языке Python.

Все криптографические компоненты разработаны самостоятельно, без использования готовых библиотек:
- Арифметика эллиптической кривой (сложение, удвоение, умножение точки на скаляр)
- Расширенный алгоритм Евклида (обратный элемент по модулю)
- Хэш-функция ГОСТ Р 34.11-2012 (Streebog-256) – полностью собственная реализация по RFC 6986

📦 Возможности
- Генерация ключевой пары (секретный ключ d, открытый ключ Q = d·P)
- Подписание произвольного файла (с вычислением хэша Streebog-256)
- Проверка подписи по файлу сообщения, файлу подписи и открытому ключу
- Консольный интерфейс для работы с файлами
- Полное соответствие алгоритмам ГОСТ Р 34.10-2012

🧠 Структура проекта

⚙️ Установка и запуск
1.Склонируйте репозиторий или скачайте все файлы в одну папку:
git clone https://github.com/your-repo/gost34.10-2012.git
cd gost34.10-2012
2.Запустите программу:
python main.py

🖥️ Использование
При запуске появится меню:
==================================================
ГОСТ Р 34.10-2012 – Электронная подпись
1. Сгенерировать ключевую пару
2. Подписать файл
3. Проверить подпись
4. Выход
Выберите действие:


1. Генерация ключей
Создаёт два файла:
- private.key – секретный ключ d (целое число)
- public.key – открытый ключ Q (две строки: x-координата, y-координата)

2. Подпись файла
Запрашивает:
- путь к файлу, который нужно подписать
- путь к файлу с закрытым ключом (private.key)
- имя файла для сохранения подписи

Программа:
1.Вычисляет хэш файла (Streebog-256)
2. Генерирует случайное k
3. Вычисляет r и s по алгоритму ГОСТ Р 34.10-2012
4. Сохраняет подпись в указанный файл (две строки: r и s)

3. Проверка подписи
Запрашивает:
- путь к файлу сообщения
- путь к файлу подписи
- путь к файлу с открытым ключом (public.key)
Выводит: ПОДПИСЬ ВЕРНА или ПОДПИСЬ НЕВЕРНА.

🔧 Параметры схемы (учебные)
В текущей реализации используются тестовые параметры из примера к ГОСТ Р 34.10-2012:
p = 97
a = 9
b = 3
q = 47
P = (-8, 1)   # (89, 1) по модулю 97
Эти параметры подходят для демонстрации и отладки.
Для реального применения необходимо заменить их на большие простые числа (256 или 512 бит) и корректно сгенерированную кривую.

- Хэш-функция Streebog-256 протестирована на тестовых векторах из RFC 6986.
